# 故障排除：主站跳转链接问题

## 问题描述

**时间：** 2026-04-08

**现象：**
- 从主站 https://www.oreocolor.xyz 点击"官网合集"和"色觉体验工具"卡片
- **手机 Safari**：打开新窗口显示空白页
- **手机 Chrome**：打开新窗口显示 `about:blank`
- **电脑（连 VPN）**：跳转到旧域名 `https://corp-site-template.vercel.app/` 和 `https://color-v2.vercel.app/`

**但是：**
- ✅ 直接访问 `https://corp.oreocolor.xyz` 可以正常打开（不用 VPN）
- ✅ 直接访问 `https://color.oreocolor.xyz` 可以正常打开（不用 VPN）
- ✅ DNS 配置正确
- ✅ 子域名在 Vercel 项目中已正确添加

---

## 原因分析

### 根本原因
**Vercel CLI 本地部署覆盖了 GitHub 自动部署**

### 详细过程
1. **早期部署使用了 `vercel --prod --force`**
   - 这个命令从**本地文件**部署
   - 绕过了 GitHub 的自动部署流程
   - 导致线上代码和 GitHub 仓库不一致

2. **后续修改代码并推送到 GitHub**
   ```bash
   git add index.html
   git commit -m "fix: 更新链接为子域名"
   git push
   ```
   - GitHub 上的代码是正确的 ✅
   - 但 Vercel 还在用旧的本地部署版本 ❌

3. **检查本地文件是正确的**
   ```bash
   grep "corp.oreocolor.xyz" index.html
   # 输出：<a href="https://corp.oreocolor.xyz/" class="project-card">
   ```
   - 本地代码正确 ✅
   - 构建后文件正确 ✅
   - GitHub 仓库代码正确 ✅

4. **但线上代码还是旧的**
   - 用户查看网页源代码：`<a href="https://color-v2.vercel.app"`
   - 说明 Vercel 没有从 GitHub 拉取最新代码

---

## 解决方案

### 方案 1：在 Vercel 控制台手动触发部署（推荐）⭐

**步骤：**

1. **打开 Vercel 项目页面**
   ```
   https://vercel.com/oreeeeeomaxs-projects/my-portfolio/deployments
   ```

2. **找到最新的 GitHub 部署**
   - 查找有 GitHub 图标的部署（不是 CLI 部署）
   - CLI 部署会显示 "CLI" 标签
   - GitHub 部署会显示提交信息

3. **点击右侧的 "..." 按钮**
   - 选择 "Redeploy"（重新部署）

4. **确认部署**
   - Vercel 会从 GitHub 拉取最新代码
   - 重新构建并部署

5. **等待 2-3 分钟**
   - 部署完成后测试

---

## 预防措施

### 1. 避免使用 CLI 本地部署

**❌ 错误做法：**
```bash
vercel --prod --force  # 从本地部署
```

**✅ 正确做法：**
```bash
git add .
git commit -m "message"
git push  # 推送到 GitHub，让 Vercel 自动部署
```

### 2. 必须使用 CLI 部署时的正确步骤

如果必须使用 CLI 部署（比如紧急修复）：

```bash
# 1. 先提交到 GitHub
git add .
git commit -m "紧急修复"
git push

# 2. 等待 Vercel 自动部署（1-2 分钟）
# 或者

# 3. 使用 vercel 命令（会从 GitHub 拉取）
vercel --prod
```

### 3. 检查部署方式

**检查部署是否来自 GitHub：**
```bash
vercel ls --prod
```

**输出示例：**
```
Age     Deployment                                                          Status
1h      my-portfolio-xxx.vercel.app (from git)                             Ready     ✅ GitHub
30m     my-portfolio-yyy.vercel.app (from cli)                             Ready     ❌ CLI
```

- **GitHub 部署**：显示提交信息和 GitHub 图标 ✅
- **CLI 部署**：显示 "CLI" 标签 ⚠️

## 技术细节

### Vercel 部署方式对比

| 方式 | 命令 | 代码来源 | 优先级 | 推荐使用 |
|------|------|----------|--------|----------|
| GitHub 自动部署 | `git push` | GitHub 仓库 | 高 | ✅ 推荐 |
| Vercel 控制台 | 点击 "Redeploy" | GitHub 仓库 | 高 | ✅ 推荐 |
| CLI 部署 | `vercel --prod` | 本地文件 | 低（会覆盖） | ⚠️ 谨慎使用 |

### 部署优先级问题

**问题：** CLI 部署会覆盖 GitHub 自动部署

**原因：**
- GitHub 自动部署由 Git webhook 触发
- CLI 部署直接上传本地构建文件
- 两者没有关联，CLI 部署会"跳过" GitHub

**结果：**
- GitHub 仓库：新代码 ✅
- 线上网站：旧代码（来自 CLI）❌
